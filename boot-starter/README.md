# boot-starter 模块

## 与 Framework 模块区别
#### 1. 只有部分模块适用的公共代码，放在boot-starter，按需引用
#### 2. 全部模块都适用的公共代码，放在framework，减少引用工作
#### 3. 本质上全部模块都适用的公共代码也可以放在boot-starter，然后在framework中引用boot-starter

## 简介

Boot Starter 模块是一个基于 Spring Boot 的自动化配置框架，旨在为微服务项目提供一系列开箱即用的 Starter 组件。该模块包含多个子模块，每个子模块都针对特定的功能领域提供自动配置和简化集成。

## 模块结构

```
boot-starter/
├── datasource/          # 数据源 Starter 组件
├── token/               # Token 管理 Starter 组件
└── encryptbody/         # 请求体加解密 Starter 组件
```

## 设计理念

### 1. 开箱即用
所有 Starter 组件都遵循 Spring Boot 的自动配置理念，只需添加依赖即可使用，无需额外配置。

### 2. 松耦合
各子模块之间相互独立，可以根据项目需求选择性引入，避免不必要的依赖。

### 3. 易扩展
提供了丰富的扩展点，允许开发者根据实际需求自定义配置和功能。

### 4. 统一规范
通过统一的接口和规范，降低了团队成员的学习成本和维护成本。

## 最佳实践

### 1. 按需引入
根据项目实际需求引入相应的 Starter 组件，避免引入不必要的依赖。

### 2. 配置优先级
建议通过外部配置文件覆盖默认配置，而非修改源代码。

### 3. 版本管理
建议在父项目的 `pom.xml` 中统一管理各 Starter 组件的版本号。
